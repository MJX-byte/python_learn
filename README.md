# git_learn
this repository is about git_learn,the content in the repository is about how to use git in the computer
1.学习资料来源及相关链接:
   在学习git的过程中，我使用了大语言模型（LLM）进行辅助，包括但不限于chatgpt，gemini，copilot，以及国产大模型豆包，我向
它们询问与git相关的基础知识点，在大模型的辅助下我成功创建了自己本地以及远程的git仓库，同时也学到了大量的与git相关的命令，包括
但不限于pull，push，git init,git add .,git status等。

2.实践流程（如安装、配置、使用过程）:
    我通过在浏览器上搜索git官方软件进行了git的安装操作，同时也通过一系列命令实现了git在电脑本地的部署以及与远程仓库的连接，
配置与使用过程中则大量使用git相关命令来进行本地git仓库的初始化（git init）以及本地代码向远程仓库的push操作和从远程仓库pull
到本地git仓库的操作。

3.对每次提交的主要内容进行简要说明:
     对于提交的代码内容，主要有两部分，分别为分支一（origin main）和分支二（origin second_branch），两个分支均是我学习
python基础知识过程中使用的代码，其中分支一主要是我在听网课过程中跟着老师练习的代码内容，其中分支二主要是python具体实践项目
的内容，通过这些内容能进一步加强我对python语法的理解以及对python这门编程语言的理解。

4.遇到的问题及解决方法（不少于2个):
     一：在将代码从本地上传到远程仓库的时候我第一次使用的是github上我的仓库的网址（即https形式），但是却一直无法上传成功，最后
在大模型的帮助下，我使用ssh的连接方式成功将代码从本地上传到远程github仓库中。
     二：我遇到了无法将本地代码上传到已包含内容的远程仓库的问题，经了解，这是因为远程仓库中的内容与本地的内容存在冲突，故而无法
直接上传代码，之后我通过强制性命令行（--force）将我远程仓库中已有的内容覆盖，成功实现了本地代码的上传。

5.Git学习心得
      git在编程协作过程中不可或缺，其意义极其巨大，能大大帮助我们提高程序编写的效率以及对代码的回溯操作，可以帮助我们进一步厘清
编程思路，同时也不用害怕代码出错，因为我们随时都可以回溯并进行修改，可谓意义巨大。
